============================================================
WHITEPAPER / eBOOK PROMPT
============================================================

BRAND DETECTION: Follow _Brand_Detection_Rules.txt before generating.

Once the brand is confirmed, use the template below.

------------------------------------------------------------
TEMPLATE
------------------------------------------------------------

Write a [CONTENT TYPE: Whitepaper / eBook] for [BRAND NAME], [BRAND DESCRIPTION], on the topic "[TOPIC]". The document should be [WORD COUNT e.g. 1500-2500 words for whitepaper / 2000-3500 words for eBook]. Write in an authoritative, research-backed tone matching the brand's market register per _Brand_Detection_Rules.txt rule 3 (default: professional B2B) targeting [TARGET AUDIENCE]. Structure the document with an executive summary, a clearly defined problem statement, data-backed insights across 4-6 sections with subheadings, actionable recommendations, and a conclusion with a CTA linking to [URL]. Include at least 4-6 credible statistics or research references. Naturally position [BRAND NAME] as a thought leader without being promotional. Do not use em dashes. Avoid passive voice.

Reference URL (if any): [INSERT URL]
Content Type: [Whitepaper / eBook]
Intended Gate: [Gated lead magnet / Ungated thought leadership]
Voice/Tone: [formal B2B / conversational B2B / first-person / brand account — default: professional B2B]

------------------------------------------------------------
DOCUMENT STRUCTURE
------------------------------------------------------------

1. Cover / Title
2. Executive Summary (150-200 words)
3. The Problem / Challenge (with data)
4. Section 1 - Insight or Theme
5. Section 2 - Insight or Theme
6. Section 3 - Insight or Theme
7. Section 4 - Insight or Theme
8. Section 5 - Insight or Theme (optional)
9. Section 6 - Insight or Theme (optional)
10. Actionable Recommendations (3-5 bullet points)
11. Conclusion with CTA
12. References

------------------------------------------------------------
OUTPUT FORMAT
------------------------------------------------------------

Full document in plain text or markdown with clearly labelled sections.

Heading format: H1 for title, H2 for main sections, H3 for subsections — use markdown heading syntax (#, ##, ###).
Each main section must be separated by a horizontal rule (---).
Table of Contents must be generated as the second element after the title.

Save to output/Whitepaper_eBook/

------------------------------------------------------------
RULES
------------------------------------------------------------

- Executive summary must stand alone as a readable summary
- Minimum 4-6 stats or research references
- Every stat or research reference must name the source, organization, and year - never "studies show" or "research suggests" without attribution
- Recommendations must be specific and actionable: 3–5 bullet points, each stating a specific action + the expected outcome. Example: 'Replace post-training surveys with 30-day behavior observation checklists — this closes the completion-vs-application gap identified in Section 3.' Avoid: 'Invest in better training technology.'
- No em dashes anywhere
- No passive voice
- Brand positioning is subtle, not promotional
- SELF-CHECK: After drafting, count total words. Whitepaper target: 1,500–2,500. eBook target: 2,000–3,500. If under the minimum, expand the Recommendations section. If over the maximum, trim the Introduction and Limitations sections.
- BATCH DEDUPLICATION: When generating multiple whitepapers or eBooks in one session, no two may share the same executive summary hook, the same primary data source, or the same CTA action. Vary the central argument and the evidence base across documents.
- Output saved to output/Whitepaper_eBook/

------------------------------------------------------------
FAILURE MODES TO AVOID
------------------------------------------------------------

1. Recommendations stated as generic advice without a specific action — 'invest in training technology' fails; name the specific action and its expected outcome.
2. Word count under minimum due to skipped Methodology or Limitations section — include both sections in every document.
3. Executive summary that repeats the introduction instead of summarising findings — the executive summary must contain the findings, not the framing.
4. Sources listed without year of publication — every reference must carry an author or organisation name and a year.
5. eBook formatted as plain text without section headings — use markdown heading syntax for all sections.
